## üí° Problem-Solving Strategy

**1. Understand the Problem**
- Read carefully
- Ask clarifying questions
- Identify inputs/outputs
- Consider edge cases

**2. Plan Your Approach**
- Think out loud
- Discuss different approaches
- Analyze time/space complexity
- Choose best approach

**3. Code**
- Start with pseudocode if helpful
- Write clean, readable code
- Use meaningful variable names
- Add comments for complex logic

**4. Test**
- Test with provided examples
- Test edge cases
- Walk through code line by line
- Fix bugs if found

**5. Optimize**
- Discuss time/space complexity
- Identify bottlenecks
- Suggest improvements
- Consider trade-offs

## üó£Ô∏è Common Interview Questions

### Technical Questions

**Q: What's the difference between == and .equals()?**
A: `==` compares references, `.equals()` compares content

**Q: What's the difference between ArrayList and LinkedList?**
A: ArrayList uses array (fast access, slow insertion/deletion in middle), LinkedList uses nodes (slow access, fast insertion/deletion)

**Q: Explain polymorphism with an example**
A: Same interface, different implementations. Parent reference can point to child objects.

**Q: What's the difference between abstract class and interface?**
A: Abstract class can have implementation, interface is purely contract (before Java 8)

### Behavioral Questions

Use STAR method:
- **Situation**: Set the context
- **Task**: Describe your responsibility
- **Action**: Explain what you did
- **Result**: Share the outcome

**Examples:**
- "Tell me about a challenging bug you fixed"
- "Describe a time you worked in a team"
- "Tell me about a project you're proud of"
