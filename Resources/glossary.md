# Java Glossary

Complete reference of Java terms and concepts. Use Ctrl+F (Cmd+F) to search.

## A

**Abstract Class**: A class that cannot be instantiated and may contain abstract methods that must be implemented by subclasses.

**Abstract Method**: A method declared without an implementation, must be overridden in subclasses.

**Access Modifier**: Keywords (public, private, protected, package-private) that control visibility of classes, methods, and fields.

**Algorithm**: Step-by-step procedure for solving a problem or performing a task.

**API (Application Programming Interface)**: Set of methods and tools for building software and applications.

**Argument**: Actual value passed to a method when calling it.

**Array**: Fixed-size data structure that holds elements of the same type.

**ArrayList**: Dynamic array that can grow or shrink in size, part of Java Collections Framework.

**Assignment**: Giving a value to a variable using the `=` operator.

**Autoboxing**: Automatic conversion between primitive types and their wrapper classes.

## B

**Binary**: Base-2 number system using only 0 and 1.

**Block**: Code enclosed in curly braces `{}`.

**Boolean**: Primitive type that holds either `true` or `false`.

**Break**: Statement that exits a loop or switch statement immediately.

**Bug**: Error or flaw in code that causes incorrect behavior.

**Bytecode**: Intermediate code that JVM executes, platform-independent.

## C

**camelCase**: Naming convention where first word is lowercase and subsequent words are capitalized (e.g., `myVariable`).

**Case-Sensitive**: Distinguishes between uppercase and lowercase letters (`Age` ≠ `age`).

**Cast**: Explicit type conversion (e.g., `(int) 3.14`).

**Char**: Primitive type that holds a single character.

**Class**: Blueprint for creating objects, defines fields and methods.

**Collections Framework**: Set of interfaces and classes for handling groups of objects.

**Comment**: Text in code that is ignored by compiler, used for documentation.

**Compilation**: Process of converting source code to bytecode.

**Compiler**: Program (`javac`) that translates Java source code to bytecode.

**Concatenation**: Joining strings together using `+` operator.

**Conditional**: Statement that executes code based on a condition (if, else, switch).

**Constant**: Variable whose value cannot change, declared with `final`.

**Constructor**: Special method that initializes objects when created.

**Continue**: Statement that skips rest of current loop iteration and continues with next.

## D

**Data Type**: Specifies the type of data a variable can hold (int, double, String, etc.).

**Debugging**: Process of finding and fixing errors in code.

**Declaration**: Statement that creates a variable (e.g., `int age;`).

**Default**: Keyword used in switch statements for unmatched cases.

**Do-While Loop**: Loop that executes at least once, then checks condition.

**Double**: Primitive type for decimal numbers with double precision (64-bit).

**Dynamic Binding**: Process of linking method call to method implementation at runtime.

## E

**Encapsulation**: OOP principle of bundling data and methods together and hiding internal details.

**Enhanced For Loop**: For-each loop for iterating over arrays or collections.

**Entry Point**: Starting point of program execution (main method).

**Enum**: Special class type that represents a group of constants.

**Equals Method**: Method to compare object contents (not references).

**Error**: Problem that occurs during program execution.

**Exception**: Event that disrupts normal program flow, can be caught and handled.

**Expression**: Combination of values, variables, and operators that produces a result.

## F

**Field**: Variable declared inside a class (instance variable or static variable).

**Final**: Keyword meaning "cannot be changed" (constants, prevent override/extend).

**Float**: Primitive type for decimal numbers with single precision (32-bit).

**For Loop**: Loop that repeats code a specific number of times.

**Framework**: Reusable software platform providing common functionality.

**Function**: See Method.

## G

**Garbage Collection**: Automatic memory management that removes unused objects.

**Generic**: Feature allowing classes/methods to work with different types while providing type safety.

**Getter**: Method that returns the value of a private field.

**GUI (Graphical User Interface)**: Visual way for users to interact with programs.

## H

**HashMap**: Collection that stores key-value pairs, allows fast lookup by key.

**HashSet**: Collection that stores unique elements with no specific order.

**Heap**: Memory area where objects are stored.

**Hierarchy**: Tree-like structure of classes related by inheritance.

## I

**IDE (Integrated Development Environment)**: Software for writing, testing, and debugging code (IntelliJ, VS Code).

**Identifier**: Name given to variable, method, class, etc.

**If Statement**: Conditional statement that executes code if condition is true.

**Immutable**: Object whose state cannot be changed after creation (like String).

**Implementation**: Code that defines what a method or interface does.

**Import**: Statement that makes classes from other packages available.

**Increment**: Increase value by one (`++` operator).

**Index**: Position of element in array or string (starts at 0).

**Inheritance**: OOP principle where class acquires properties/methods from parent class.

**Initialization**: Assigning initial value to variable.

**Instance**: Specific object created from a class.

**Instance Method**: Method that belongs to object, not class.

**Instance Variable**: Variable unique to each object.

**Instantiation**: Creating an object from a class using `new`.

**Int**: Primitive type for integers (whole numbers).

**Interface**: Contract specifying methods that implementing classes must provide.

**Interpreter**: Program that executes code line by line.

**Iteration**: One pass through a loop.

**Iterator**: Object for traversing collections.

## J

**JAR (Java Archive)**: File format for distributing Java programs and libraries.

**Java**: High-level, object-oriented programming language.

**JavaDoc**: Tool and format for generating API documentation from comments.

**JDK (Java Development Kit)**: Tools for developing Java applications.

**JIT (Just-In-Time) Compiler**: Compiles bytecode to native machine code at runtime.

**JRE (Java Runtime Environment)**: Software needed to run Java applications.

**JVM (Java Virtual Machine)**: Executes Java bytecode, provides platform independence.

## K

**Keyword**: Reserved word with special meaning in Java (if, while, class, etc.).

## L

**Library**: Collection of pre-written code that provides functionality.

**Literal**: Fixed value in code (e.g., `42`, `"hello"`, `true`).

**Local Variable**: Variable declared inside a method, only accessible within that method.

**Logic Error**: Bug where code runs but produces incorrect results.

**Long**: Primitive type for large integers (64-bit).

**Loop**: Structure that repeats code multiple times.

## M

**Main Method**: Entry point where Java program starts: `public static void main(String[] args)`.

**Member**: Field or method of a class.

**Memory Leak**: Memory not properly released, causing program to consume more memory over time.

**Method**: Block of code that performs a specific task, can be called multiple times.

**Method Overloading**: Defining multiple methods with same name but different parameters.

**Method Overriding**: Redefining parent class method in subclass.

**Method Signature**: Method name plus parameter types.

**Modulo**: Remainder operation (`%` operator).

**Mutability**: Ability to change object after creation.

## N

**Nested Loop**: Loop inside another loop.

**New**: Keyword used to create objects.

**Null**: Special value indicating "no object" or "no value".

**NullPointerException**: Error when trying to use null reference.

## O

**Object**: Instance of a class with its own state and behavior.

**Object-Oriented Programming (OOP)**: Programming paradigm based on objects and classes.

**Operand**: Value that an operator acts upon.

**Operator**: Symbol that performs operation (+, -, *, /, ==, etc.).

**Override**: Redefining inherited method in subclass.

## P

**Package**: Namespace for organizing related classes and interfaces.

**Parameter**: Variable in method declaration that receives argument value.

**PascalCase**: Naming convention where each word starts with capital letter (e.g., `MyClass`).

**Pass-by-Value**: Java's parameter passing mechanism (copies values, not references).

**Polymorphism**: Ability of objects to take many forms, central OOP concept.

**Primitive Type**: Basic data type built into Java (int, double, boolean, char, etc.).

**Private**: Access modifier restricting access to within the same class.

**Procedure**: See Method.

**Protected**: Access modifier allowing access within package and subclasses.

**Public**: Access modifier allowing access from anywhere.

## Q

**Queue**: Collection following First-In-First-Out (FIFO) principle.

## R

**Recursion**: Method that calls itself to solve problem.

**Reference**: Address pointing to object in memory.

**Reference Type**: Non-primitive type (classes, interfaces, arrays).

**Return**: Statement that exits method and optionally returns value.

**Return Type**: Data type of value method returns (or `void`).

**Runtime**: Period when program is executing.

## S

**Scanner**: Class for reading user input from various sources.

**Scope**: Region where variable is accessible.

**Setter**: Method that sets/changes value of private field.

**Short**: Primitive type for small integers (16-bit).

**Signature**: See Method Signature.

**Stack**: Memory area for method calls and local variables; also LIFO data structure.

**Static**: Keyword indicating member belongs to class, not instances.

**String**: Immutable sequence of characters.

**StringBuilder**: Mutable sequence of characters for efficient string building.

**Subclass**: Class that inherits from another class (child class).

**Superclass**: Class that is inherited from (parent class).

**Switch**: Conditional statement for multiple specific cases.

**Syntax**: Rules defining valid code structure.

**Syntax Error**: Mistake in code structure preventing compilation.

**System.out**: Standard output stream for printing to console.

## T

**Ternary Operator**: Conditional operator with three operands: `condition ? true : false`.

**This**: Keyword referring to current object.

**Thread**: Independent path of execution within program.

**Throw**: Statement that explicitly throws exception.

**Throws**: Declares that method may throw exception.

**Try-Catch**: Block for handling exceptions.

**Type**: Category of data (int, String, etc.) or class.

**Type Casting**: Converting value from one type to another.

## U

**UML (Unified Modeling Language)**: Standard for visualizing system design.

**Unary Operator**: Operator with one operand (++, --, !).

**Unboxing**: Automatic conversion from wrapper class to primitive.

**Unicode**: Character encoding standard supporting international characters.

## V

**Variable**: Named storage location holding value that can change.

**Void**: Return type indicating method returns nothing.

## W

**While Loop**: Loop that continues while condition is true.

**Whitespace**: Spaces, tabs, and newlines (ignored by compiler).

**Widening**: Automatic conversion from smaller to larger type.

**Wrapper Class**: Class that wraps primitive type (Integer, Double, Boolean, etc.).

**WORA (Write Once, Run Anywhere)**: Java's platform independence principle.

## X

**XOR (Exclusive OR)**: Logical operator (^) true when operands differ.

## Common Abbreviations

- **CRUD**: Create, Read, Update, Delete
- **GUI**: Graphical User Interface
- **IDE**: Integrated Development Environment
- **I/O**: Input/Output
- **JAR**: Java Archive
- **JDK**: Java Development Kit
- **JIT**: Just-In-Time
- **JRE**: Java Runtime Environment
- **JVM**: Java Virtual Machine
- **OOP**: Object-Oriented Programming
- **UML**: Unified Modeling Language
- **WORA**: Write Once, Run Anywhere
